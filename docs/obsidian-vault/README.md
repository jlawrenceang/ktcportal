# Obsidian Vault (Knowledge Mirror)

This vault is a structured knowledge mirror of the KTC broker portal.

## Important

- Vault notes must align to runtime code/routes and migrations.
- If a vault note conflicts with runtime implementation, update the note or archive the stale content.
- Session notes are historical; core/system notes are operational references.

## Primary folders

- `01-System` — architecture, runtime target, release gate, invariants, tech stack
- `02-Cores` — per-core pages/tables/RPCs/status (Authentication, Brokers, Consignees, Job Orders, Administration)
- `04-Workflows` — operational chains
- `05-Concepts` — domain concepts and dictionaries
- `06-Sessions` — historical session logs
- `07-Memory` — current state, pending items, roadmap, milestones, waves, scale
- `09-Future` — future / non-committed ideas

See `docs/agent/obsidian-memory-policy.md` for the per-session update ritual.
